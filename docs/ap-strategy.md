# 応用情報技術者試験（AP）学習戦略

## 目的
- 応用情報技術者試験に **短期間・高効率で合格**する
- CBT化や試験形式の変更があっても、学習方針がブレないようにする

---

## 基本方針

応用情報は「全部できる人」ではなく、  
**「取る問題を決めて確実に取る人」が合格する試験**である。

- 午前は **6〜7割を安定**させる
- 午後は **選ぶ分野を固定して8割を狙う**
- 出題範囲が広いため、**分野ごとに学習の重さを変える**

---

## 重点分野・非重点分野の考え方

### 方針の要点
- **重点分野：得点源として深く学ぶ**
- **非重点分野：落とさないために浅く触れる**

すべてを同じ重さで学ばない。

---

## 重点分野（得点源）

### 対象分野
- ネットワーク
- データベース
- 情報セキュリティ
- （午前のみ）コンピュータ構成要素  
  ※ CPU / メモリ / OS など

### 基本方針
- 合格点を安定して取るための最重要分野
- **1分野ずつ、フェーズを縦に進め切る**
- AP 学習時間の中心に置く（heavy）

### 進め方（原則）
- フェーズ1 → フェーズ2 → フェーズ3 を順に進める
- 原則として **複数の重点分野を並行しない**
- 完璧を目指さず、次に進める状態になったら分野を切り替える

### フェーズ切り替えの目安
- 午後問題を見て「何を聞かれているか」が分かる
- 午前問題で用語が完全に未知ではない

### Issue 例  
- AP 午前：ネットワーク基礎理解（フェーズ1）
- AP 午後：ネットワーク過去問演習（フェーズ2）
- AP 午後：NW 弱点補強（フェーズ3）

---

## 非重点分野（落とさないための分野）

### 対象分野
- 法務
- 経営戦略
- システム戦略
- ソフトウェア開発
- アルゴリズム
- マネジメント系（PMなど）

### 基本方針
- 深追いしない
- heavy にしない
- **light 学習で広く浅く触れる**

### 進め方
- **週に1分野ペース**
- 水・日の AP復習枠を利用する

#### 水曜日（平日 light）
- 初見で触る
- 過去問 5〜10問
- 正誤は気にしない

#### 日曜日（週末復習）
- 同じ分野を軽く振り返る
- 過去問 3〜5問
- ノートを短時間見返す

### 優先順位
1. 法務  
2. 経営戦略  
3. システム戦略  
4. ソフトウェア開発  
5. アルゴリズム  
6. マネジメント系  

※ 上に行くほど短時間で点につながりやすい

### Issue 例
- AP 午前：法務 過去問5問（light）
- AP 復習：法務（今週分）

---

## 学習フェーズ

### フェーズ1：基礎構築
**目的**
- 午前の用語・構造を理解する
- 午後問題を「読める」状態にする

**やること**
- 教科書・参考書で概要把握
- 午前の基礎確認
- 午後問題を解かずに構造を読む

---

### フェーズ2：過去問反復
**目的**
- 合格ラインの問題を安定して取れるようにする

**やること**
- 午前：分野別に過去問
- 午後：年度別に過去問
- 正解率より「なぜそうなるか」を重視

---

### フェーズ3：弱点補強
**目的**
- 毎回落とすパターンを潰す

**やること**
- 間違えた分野・設問だけをピンポイントで演習
- 新しい分野には手を出さない

---

## GitHub 運用との対応

- 重点分野の学習 → `ap + heavy`
- 非重点分野の学習 → `ap + light`
- 復習 → `ap + light`

### Issue 運用ルール
- 1 Issue = 30〜90分で完了できる作業
- 完了条件は「理解」ではなく **実施した証拠**
- 実施したら Done / Close（出来・理解度は問わない）

---

## 重要な考え方（まとめ）

- 非重点分野は「できるようにする」対象ではない
- **「見たことがある」「聞いたことがある」状態を作る**
- 点を取りに行くのは重点分野だけ

> 応用情報は  
> 「全部を平均的にできる人」ではなく  
> **「取る分野を決めて確実に取る人」が合格する**
